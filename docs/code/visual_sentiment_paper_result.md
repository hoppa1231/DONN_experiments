# `visual/sentiment_paper_result.py`

## Зачем нужен этот файл

Этот файл запускает не “удобную локальную” версию Table 4, а именно
контрольный paper-style прогон.

Его задача:

- взять более буквальную архитектуру из полной Table 4;
- обучить её отдельно от старой локальной адаптации;
- сохранить отдельные PNG и JSON;
- сразу показать, насколько результат расходится с заявлением статьи.

## Что делает по шагам

1. Разбирает аргументы командной строки.
2. Вызывает `train_paper_exact_run(...)` из `src/sentiment.py`.
3. Берёт тестовые отзывы и предсказания модели.
4. Выбирает по одному отрицательному и положительному примеру.
5. Строит итоговую картинку:
   - сверху фрагменты двух отзывов,
   - снизу вероятности классов для этих двух примеров.
6. Сохраняет JSON с метриками и служебной информацией:
   - accuracy,
   - loss,
   - число параметров,
   - расхождение с параметрами из статьи,
   - история обучения.

## Почему этот файл отделён от `visual/sentiment_result.py`

Потому что это два разных сценария:

- `visual/sentiment_result.py`
  это локальная практическая версия, удобная для демонстрации;
- `visual/sentiment_paper_result.py`
  это контрольный эксперимент для проверки самой статьи.

Если бы оба режима были смешаны в одном visual-скрипте без явного разделения,
потом было бы трудно понять, какой именно результат на что отвечает.

## Какие файлы он обычно создаёт

- `artifacts/plots/fourth_work_paper_exact_summary_4k3e.png`
- `artifacts/plots/fourth_work_paper_exact_metrics_4k3e.json`

В этих файлах лежит уже не локальная аппроксимация, а проверка более строгого
Table 4 path.

## На что смотреть в JSON

Самые важные поля:

- `test_acc`
- `val_acc`
- `test_loss`
- `total_params`
- `non_embedding_params`
- `paper_reported_total_params`
- `param_gap_total_vs_paper`
- `param_gap_non_embedding_vs_paper`

Именно они позволяют быстро понять:

- вышли ли мы на заявленную точность;
- совпадает ли модель со статьёй хотя бы по масштабу параметров.

## Что важно помнить

Даже этот файл не гарантирует “абсолютно точную” копию авторского кода, потому
что статья недоопределяет некоторые детали обучения.

Но он уже намного ближе к опубликованной Table 4, чем старая версия с
sequence-ramp target'ами.
