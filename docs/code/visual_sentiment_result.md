# `visual/sentiment_result.py`

## Для чего нужен файл

Это исполняемый сценарий для Table 4.

Он делает из текстовой модели понятный showable-отчёт.

## Что особенно важно в этом файле

Если просто смотреть на одно число accuracy,
то трудно понять, что именно делает модель.

Этот файл решает проблему так:

- показывает фрагменты двух отзывов,
- показывает последовательностные выходы DONN,
- добавляет baseline для сравнения.

## Что делают основные части

### `select_negative_positive(labels)`

Ищет:

- один отрицательный пример,
- один положительный пример.

Зачем:

- чтобы визуальный отчёт обязательно показывал обе тональности.

### `_format_review(...)`

Аккуратно переносит длинный текст на несколько строк,
чтобы он не ломал фигуру.

### `plot_text_panel(...)`

Рисует верхнюю текстовую панель.

Что там показывается:

- короткий фрагмент отзыва,
- истинная метка,
- предсказанная метка.

Почему текст на графике всё равно английский:

- потому что отзывы берутся из исходного IMDB-датасета,
- это не служебный текст, а сами данные.

Что мы уже улучшили:

- служебные подписи переведены на русский,
- длина фрагмента уменьшена,
- на картинке теперь меньше лишнего англоязычного текста.

### `plot_trace_panel(...)`

Рисует нижний график для одного примера.

Что там видно:

- целевая рампа правильного класса,
- предсказанная оценка правильного класса,
- предсказанная оценка второго класса.

Зачем это нужно:

- так можно увидеть, различает ли модель классы по ходу последовательности.

### `plot_report(...)`

Собирает итоговую фигуру `2 x 2`:

- отрицательный отзыв,
- положительный отзыв,
- нижний график для отрицательного примера,
- нижний график для положительного примера.

### `main()`

Основной сценарий:

1. читает параметры,
2. обучает DONN,
3. при необходимости обучает BiLSTM baseline,
4. строит итоговую картинку,
5. сохраняет PNG и JSON.

## Почему baseline встроен прямо сюда

Потому что для Table 4 он не просто “приятное дополнение”.
Он диагностически обязателен.

Если DONN слабый, а baseline заметно лучше,
значит:

- данные и разбиение рабочие,
- проблема сидит именно в текущем DONN-переносе на текст.

Именно это и показывает наш текущий результат.
