# `visual/operators_result.py`

## Для чего нужен файл

Это исполняемый сценарий для Table 3.

Он запускает сразу две подзадачи:

- интегрирование,
- дифференцирование.

## Что делает файл по шагам

### 1. Подключает `TASKS` и `train_one_task(...)`

То есть берёт:

- список задач,
- единый training pipeline из `src/operators.py`.

### 2. `plot_task_row(...)`

Это функция, которая рисует одну строку таблицы-графика.

В одной строке есть:

- входной сигнал,
- два примера результата,
- текстовый блок с метриками.

Почему это удобно:

- одна и та же логика подходит и для интегрирования, и для дифференцирования,
- код не дублируется.

### 3. `plot_report(...)`

Здесь собирается общий рисунок `2 x 3`:

- верхняя строка — интегрирование,
- нижняя строка — дифференцирование.

Это хороший формат для сравнения,
потому что обе задачи видны в одном окне.

### 4. `main()`

Главный сценарий:

1. читает CLI-параметры,
2. по очереди запускает обе задачи,
3. сохраняет всё нужное в словарь `results`,
4. вызывает рисование фигуры,
5. сохраняет JSON с метриками.

## Почему этот файл особенно полезен

Потому что он не просто рисует DONN-предсказание,
а ещё показывает численный baseline.

Для Table 3 это очень важно:

- именно baseline помогает сразу увидеть, где DONN ещё проигрывает,
- особенно в задаче дифференцирования.
