# `src/demodulation.py`

## Для какой задачи нужен файл

Этот файл отвечает за Table 2:

- амплитудную демодуляцию.

То есть модель получает модулированный сигнал и должна восстановить исходное сообщение.

## Главная инженерная идея файла

Самое важное улучшение здесь — временной `Conv1D`-readout после осцилляторов.

Именно это исправило ситуацию, когда старая DONN-реализация видела форму сигнала,
но плохо восстанавливала правильную демодуляцию.

## Что делает файл

### `generate_demod_dataset(...)`

Что происходит:

1. создаётся временная сетка,
2. случайно выбираются частоты компонент сообщения,
3. из них собирается `message`,
4. создаётся несущая,
5. строится модулированный сигнал:
   `(1 + message) * carrier`.

Что возвращается:

- `x` — модулированный сигнал,
- `y` — исходное сообщение,
- `t` — время.

### `split_train_test(...)`

Обычное перемешивание и разбиение на train/test.

### `DONNDemodulator`

Это основная модель.

Её схема:

1. линейный frontend,
2. слой Хопфа,
3. нормализация признаков,
4. два временных `Conv1D`,
5. выходной `Conv1D`.

Зачем нужен `use_input_skip`:

- модель получает не только признаки осцилляторов,
- но и сам исходный вход,
- а также `abs(x)`.

Почему это полезно:

- демодуляция очень сильно зависит от формы входного сигнала,
- skip-путь помогает readout-части не терять эту информацию.

### `train_one_run(...)`

Что делает:

- делит данные,
- создаёт модель,
- обучает её по MSE,
- считает финальный MSE на тесте,
- возвращает:
  - метрики,
  - тестовый вход,
  - тестовую цель,
  - предсказание.

## Что важно понять новичку

Этот файл уже не просто “запускает Hopf и сразу делает выход”.

Он сначала строит oscillatory features, а потом даёт отдельной временной голове
возможность собрать демодулированный сигнал.

Именно это делает код более похожим на практическую инженерную систему,
а не на слишком жёсткую минимальную схему.
