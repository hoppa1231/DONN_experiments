# `src/operators.py`

## Для какой задачи нужен файл

Этот файл отвечает за Table 3:

- интегрирование,
- дифференцирование.

## Почему файл вообще понадобился

В старом репозитории локальные данные для этой таблицы были противоречивыми.

Проблемы были такие:

- где-то не хватало `Y.npy`,
- где-то локальные генераторы не совпадали с формулами статьи,
- из-за этого было трудно понять, проблема в данных или в модели.

Поэтому здесь данные генерируются заново прямо по формулам статьи.

## Основные части файла

### `generate_operator_dataset(...)`

Что делает:

1. создаёт случайную сумму синусоид,
2. для `integration` строит аналитический интеграл,
3. для `differentiation` строит аналитическую производную,
4. возвращает вход, цель и временную сетку.

Почему это хороший ход:

- мы точно знаем, какая должна быть правильная цель,
- нет зависимости от подозрительных старых `.npy` файлов.

### `split_train_test(...)`

Обычное случайное разбиение на train/test.

### `_numeric_prediction(...)` и `numeric_baseline(...)`

Это очень важная часть.

Что здесь происходит:

- для дифференцирования используется `np.gradient`,
- для интегрирования используется накопление площади трапецией.

Зачем это нужно:

- чтобы сравнить DONN не только со статьёй,
- но и с простым численным решением той же задачи.

Это даёт очень полезную диагностику:

- если baseline почти идеален, а DONN плохой, значит задача не сломана,
- значит проблема именно в модели или её head.

### `DONNOperatorRegressor`

Это основная модель файла.

Её схема:

1. линейный frontend,
2. слой Хопфа,
3. временной `Conv1D` readout,
4. выходной `Conv1D`.

Почему здесь снова `Conv1D`, а не просто Dense по каждому шагу:

- интегрирование и дифференцирование — это временные операторы,
- им нужен доступ к соседним точкам времени,
- pointwise-голова для этого слишком слаба.

### `_safe_scale(...)`

Небольшая служебная функция.

Она гарантирует, что масштаб для нормализации не станет нулём.

### `train_one_task(...)`

Это главный рабочий конвейер.

Что он делает:

1. генерирует данные,
2. делит их на train/test,
3. считает численный baseline,
4. нормализует train-данные,
5. обучает DONN,
6. считает:
   - `test_mse`,
   - `val_mse`,
   - `test_corr`,
   - `baseline_mse`,
7. возвращает всё нужное для построения картинки.

## Главный вывод по файлу

Этот файл показал, что:

- интегрирование воспроизводится локально довольно хорошо,
- дифференцирование раньше проваливалось в основном из-за масштаба и слабой головы,
- после нормализации и temporal readout задача стала намного лучше.
