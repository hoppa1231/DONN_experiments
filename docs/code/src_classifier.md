# `src/classifier.py`

## Для какой задачи нужен файл

Этот файл отвечает за Table 1:

- классификацию сигналов по частотным диапазонам.

## Главная мысль

Старая локальная версия плохо училась, когда задача была записана как регрессия по рампам.
Здесь реализован более удачный вариант:

- модель учится как классификатор,
- а не как регрессор по последовательности.

## Что делают основные части

### `labels_from_y(y)`

Что делает:

- берёт исходный целевой массив `Y`,
- переводит его в обычные метки классов `0` и `1`.

Почему это полезно:

- дальше можно использовать нормальную классификационную функцию потерь.

### `DONNClassifierCE`

Это основная модель файла.

Её схема:

1. линейный frontend,
2. слой Хопфа,
3. проекция в компактное скрытое пространство,
4. усреднение по времени,
5. классификационная голова.

Подробно:

- `in_r` и `in_i`:
  превращают вход в признаки для реальной и мнимой части осцилляторов.
- `hopf`:
  прогоняет сигнал через динамику осцилляторов.
- `proj`:
  делает скрытое представление компактнее.
- `pool`:
  собирает информацию по времени в один вектор.
- `head`:
  выдаёт логиты классов.

Почему это лучше старой версии:

- задача реально является классификацией,
- поэтому `cross-entropy` лучше подходит, чем MSE по рампам.

### `train_one_run(...)`

Что делает:

- фиксирует random seed,
- делит данные на train/test,
- создаёт модель,
- компилирует её,
- обучает,
- считает качество на test,
- печатает распределение классов.

Почему печатается `class_hist`:

- это быстрый контроль на схлопывание модели в один класс.

### `Metrics`

Это маленький dataclass для аккуратного возврата метрик.

Он хранит:

- `test_acc`
- `val_acc`
- `test_loss`

## Что важно понять новичку

Этот файл не рисует картинки.

Он только:

- строит модель,
- учит модель,
- возвращает численные результаты.

Картинку уже строит `visual/classifier_result.py`.
