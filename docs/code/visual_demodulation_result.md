# `visual/demodulation_result.py`

## Для чего нужен файл

Это исполняемый сценарий для Table 2.

Он нужен не для разработки самой модели,
а для получения финального showable-результата:

- обучить модель,
- показать картинку,
- сохранить метрики.

## Что делает файл

### 1. Подключает код из `src/demodulation.py`

Это значит, что саму логику модели он не дублирует.
Он просто вызывает уже готовые функции.

### 2. Строит красивый отчёт

Функция `plot_report(...)` рисует:

- исходное сообщение `m(t)`,
- модулированный вход `M(t)`,
- два примера восстановления сообщения.

Почему именно так:

- это самый быстрый визуальный ответ на вопрос
  “модель реально демодулирует или нет?”

### 3. `main()`

Здесь:

1. читаются параметры запуска,
2. генерируется синтетический датасет,
3. запускается обучение,
4. собираются метрики,
5. строится PNG,
6. сохраняется JSON.

## Почему этот файл не стоит перегружать логикой

Потому что его цель другая:

- `src/demodulation.py` должен быть переиспользуемым,
- `visual/demodulation_result.py` должен быть простым и удобным для показа.

Это хорошее разделение обязанностей:

- логика отдельно,
- презентация отдельно.
