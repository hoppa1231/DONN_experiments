# `src/HopfLayer.py`

## Главная идея файла

Это центральный файл всего проекта.

Он реализует слой Хопфа, то есть набор осцилляторов, которые получают входной сигнал
и развиваются по времени по дифференциальным уравнениям.

Если говорить совсем просто, то этот файл делает из обычного входа
осциллирующее скрытое представление.

## Что здесь происходит по частям

### `set_seed(seed)`

Что делает:

- фиксирует случайность в NumPy,
- фиксирует случайность в TensorFlow.

Зачем:

- чтобы прогоны были воспроизводимыми,
- чтобы результаты не “прыгали” просто из-за нового random seed.

### `_real_part(r, phi)` и `_imag_part(r, phi)`

Что делают:

- превращают внутреннее представление осциллятора из формы
  “радиус + фаза”
  в форму
  “действительная часть + мнимая часть”.

Формула:

- действительная часть = `r * cos(phi)`
- мнимая часть = `r * sin(phi)`

Зачем это нужно:

- дальше модели удобнее работать именно с такими признаками.

### `_hopf_rollout(...)`

Это сердце файла.

Именно здесь по шагам времени считается поведение осцилляторов.

Что функция получает:

- `x_r`, `x_i` — входные сигналы для действительной и мнимой части,
- `omegas` — собственные частоты осцилляторов,
- `num_steps` — сколько временных шагов нужно пройти,
- `dt` — размер одного временного шага,
- `mu`, `beta` — параметры динамики,
- `input_scale` — насколько сильно вход влияет на осцилляторы.

Что она делает:

1. создаёт начальное состояние:
   - радиус `r_t = 1`
   - фаза `phi_t = 0`
2. идёт по каждому временному шагу,
3. обновляет радиус и фазу по формулам Эйлера,
4. сохраняет историю всех состояний,
5. возвращает траектории по всему времени.

Почему это важно:

- здесь DONN перестаёт быть обычной нейросетью и начинает вести себя как динамическая система.

### `HopfLayer`

Это удобная Keras-обёртка над `_hopf_rollout`.

Что хранит слой:

- число осцилляторов,
- число шагов времени,
- диапазон частот,
- шаг интегрирования,
- параметры динамики,
- коэффициент влияния входа.

Что изменилось в нашем проекте:

- раньше частоты осцилляторов были просто константой,
- теперь они могут быть `trainable`,
- это нужно для задач вроде Table 4, где статья прямо говорит, что частоты обучаются.

### `self.omegas`

Что это:

- набор собственных частот осцилляторов.

Что мы сделали:

- превратили их в `add_weight(...)`,
- чтобы TensorFlow мог оптимизировать их вместе с остальными параметрами.

## Как читать этот файл новичку

Не нужно сразу вникать в каждую формулу.

Достаточно понять такую цепочку:

1. входной сигнал поступает в слой,
2. слой запускает осцилляторы,
3. осцилляторы идут по времени шаг за шагом,
4. на выходе получаются временные признаки,
5. этими признаками уже пользуются остальные модели.

## Почему этот файл самый важный в репозитории

Потому что:

- `classifier.py` зависит от него,
- `demodulation.py` зависит от него,
- `operators.py` зависит от него,
- `sentiment.py` зависит от него.

Если здесь ошибка, то ошибка автоматически размазывается по всем таблицам.
