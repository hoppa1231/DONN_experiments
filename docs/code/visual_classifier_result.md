# `visual/classifier_result.py`

## Для чего нужен файл

Это исполняемый сценарий для Table 1.

Если `src/classifier.py` — это двигатель,
то этот файл — водительское место:

- он запускает обучение,
- получает результаты,
- строит финальную картинку,
- сохраняет JSON с метриками.

## Что делает по шагам

### 1. Готовит пути и импорты

В начале файл добавляет корень проекта в `sys.path`.

Зачем:

- чтобы можно было запускать скрипт прямо командой:

```powershell
.\.venv\Scripts\python visual\classifier_result.py
```

### 2. Делит данные на train/test

Функция `split_train_test(...)`:

- перемешивает индексы,
- отделяет test-часть,
- возвращает train и test.

### 3. Выбирает по одному примеру каждого класса

Функция `select_two_classes(...)` нужна для визуализации.

Она специально ищет:

- один пример класса 0,
- один пример класса 1.

### 4. Получает амплитуды осцилляторов

Функция `get_oscillator_amplitudes(...)` вручную проходит часть модели,
чтобы достать внутреннюю амплитуду осцилляторов Хопфа.

Зачем:

- это делает картинку не просто “вход/выход”,
- а ещё и показывает, какие частоты активировались внутри сети.

### 5. Превращает логиты в рампы

`logits_to_ramps(...)` делает из вероятностей классов
визуально понятные последовательности, похожие на целевые рампы.

Это удобно именно для графика.

### 6. Рисует итоговую фигуру

`plot_report(...)` строит три блока:

- входные сигналы двух классов,
- целевые и предсказанные рампы,
- профиль амплитуд осцилляторов по частотам.

### 7. `main()`

Это главный сценарий:

1. читает параметры командной строки,
2. грузит `X.npy` и `Y.npy`,
3. учит модель,
4. считает метрики,
5. строит график,
6. сохраняет PNG и JSON.

## Почему файл полезен новичку

Он показывает полный жизненный цикл эксперимента в одном месте:

- загрузка,
- обучение,
- оценка,
- визуализация,
- сохранение.
